\section{Introduction}
Hierarchical clustering is a way to recursively partition data into clusters. Given a weighted graph $G = (V, E, c)$, where $c\colon E\to \mathbb{N}$, the goal is to find a clustering tree $T$, in which the root node $u$ corresponds to a subset of edges $S_u$ to be deleted first. Each child node $v$ is then associated with a connected component of $G\setminus S_u$. The process continues recursively until all edges are deleted and the remaining vertices are singletons. For each node $u$, define $G_u$ to be the graph in which $u$ is the root of the subtree. The \textit{Dasgupta's clustering objective} is defined as $\COST_G\br{T} = \sum_{u\in V\br{T}} c(S_u) \cdot |V(G_u)|$\footnote{Note that the usual way of defining the Dasgupta's clustering objective is $\COST_G\br{T} = \sum_{uv\in E} c(uv) \cdot |\ell_T\br{u,v}|$, where $\ell_T\br{u,v}$ denotes the smallest subtree containing both $u$ and $v$, but these are equivalent.}. The problem of finding a hierarchical clustering tree with minimum cost is known as the \textsc{Hierarchical Clustering} problem. Among possible interpretation of this clustering cost is the average summaric cost of cuted edges on a path from root of $T$ to a node of $T$ denoting the last cut before a vertex chosen uniformly at random belongs to a tree cluster.

In this work we focus on a variant of the above problem, in which we modify the halting condition of the recursive process. Instead of stoping when all edges are deleted, the cluster tree stops when the remaining cluster is a tree. We call this problem \textsc{Hierarchical Acyclic Clustering}. This problem is a hierarchical version of the \textsc{Feedback Edge Set}. The goal is again to minimize $\COST_G\br{T} = \sum_{u\in V\br{T}} c(S_u) \cdot |V(G_u)|$. Since any Hierarchical Clustering tree is also a Hierarchical Acyclic Clustering tree this problem is a relaxation of the \textsc{Hierarchical Clustering}.

\subsection{Motivations and Applications}
Hierarchical clustering is a clusterization technique far richer in its structure and applications. Since such a object encodes a hierarchical decomposition of data, one may treat each node of $T$ as a way to encode an iteratively refined way of partitioning the domain. For example any phylogenetic tree is a clusterization as well as file system, management hierarchy in a company, socio-political divisions of people and many more. The motivation of relaxing the halting condition of the clustering is due to the fact that trees are among simplest classes of graphs and therefore most of their properties are well understood. Moreover, many problems which are hard on general graphs can be solved efficiently on trees. Since the usual clustering tree consists of cutting all edges of the input graph, such an object might becomes too large to store. Since many real life networks are sparse, it is sensible to ask whether we can get a more compact representation of the hierarchical decomposition of the data. 

Among possible usages of this problem, we can build a heuristic for the \textsc{Hierarchical Clustering} problem by solving the \textsc{Hierarchical Acyclic Clustering}. The idea is as follows: we firstly run the algorithm for \textsc{HAC} to get a clustering tree $T$, such that each occurring cluster $A$ is a tree. Then, for each such $A$ we run any algorithm for \textsc{HC} on $A$ to find the remaining levels of clusterization. Since \textsc{HC} of a tree with a constant factor error can be found in polynomial time \cite{ApproximateHierarchicalClusteringViaSparsestCutAndSpreadingMetrics} the most expensive part of the cost of the clustering tree will be the cost of the \textsc{HAC}. Therefore, the better approximation algorithm we have for \textsc{HAC}, the better the performance of the heuristic becomes. 
Note that one can also use this heuristic dynamically, in the sense that the \textsc{HAC} is stored statically and whenever any cluster needs to be refined, we can run the \textsc{HC} algorithm on the corresponding tree cluster. This way we can get a dynamic clustering tree which is not stored in its entirety but rather built on demand.

We also observe that despite similariies, the objective of \textsc{HAC} can be much lower then the objective of \textsc{HC}. Let $G$ by a star with one of the vertices of the star being replaced by a triangle. The optimal clustering tree for \textsc{HC} will have cost $\Theta\br{n^2}$, since at least $(n-2)/2$ vertices have need to have at least $(n-2)/2$ edges cut before they become singletions. On the other hand, the optimal clustering tree for \textsc{HAC} will have cost $\Theta\br{n}$, since the  optimal clustering consists of cutting one edge of the triangle. This suggest that \textsc{HAC} is a significantly different from \textsv{HC}.
